# 割り箸カード v100 魔法少女ドロー・ターン移行修正版

## 原因

貪欲・憤怒・幸福のドロー処理が、
存在しない `drawOne()` 関数を呼び出していた。

貪欲が付いているプレイヤーのターン開始時にJavaScriptエラーとなり、
`startTurn()` が途中で停止するため、直前のターン終了処理まで
止まったように見える状態になっていた。

## 修正

すべて共通ドロー処理 `drawCard()` へ統一。

対象：

- 貪欲：ターン開始時に2枚引き、ランダムに2枚捨てる
- 憤怒：ターン開始時に追加で1枚引く
- 幸福：攻撃後に2枚引く

貪欲は実際に引けた枚数と捨てた枚数をログへ表示する。
山札切れの場合は通常の疲弊処理を通る。

## ターン終了処理の確認

- 通常CPU戦の `endTurn()` → `startTurn()` → `cpuTurn()` を確認
- オンライン戦の即時同期分岐を確認
- チュートリアル専用停止条件は
  `battleMode === "tutorial"` かつ専用フラグが有効な時だけ動作
- 通常対戦への条件漏れは確認されなかった

今回の停止原因はチュートリアル分岐ではなく、
次ターン開始時の未定義ドロー関数エラー。
