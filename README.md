# 割り箸カード v70 指令安定化修正版

## 指令の加護

### 調査結果
従来版でも軽減計算は存在していたが、次の点により効果がないように見えることがあった。

- 最低1本のため、1本を加える攻撃・効果は見た目上1本のまま
- 数値が変わらなかった場合、ログが出なかった
- 達成数の記録が「次の相手ターン」以外にも残り得た

### 修正
- 指令達成数を「加護用の有効軽減値」として別管理
- 自分のターン終了時に軽減値を記録
- 次の相手ターン中だけ有効
- 次の自分ターン開始時に失効
- 最低1本で数字が変わらない場合も、加護が働いたことをログ表示
- 軽減値を記録した時点でもログ表示

## 対象指定がない指令

### 原因
通常ドロー以外の経路で元の指令カードIDが直接手札へ入ると、
右・左の指定情報を持たないまま表示される可能性があった。

### 修正
- 手札へ入る際に指令を必ず派生カードへ変換
- 手札表示直前にも全指令を検査・補完
- ターン終了時の指令判定直前にも再検査
- 指定データが壊れた派生カードも自動で再抽選
- 都市の意志・意志の奔流で移動した指令も正規化

## 表記
- `R` → `右`
- `L` → `左`
- `指令：指定攻撃［右］`
- `指令：対象指定［右→左］`
- カード本文の指定表示も右・左へ統一
